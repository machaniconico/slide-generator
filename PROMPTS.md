# スライド生成用プロンプト集

INSTRUCTIONS.md と一緒にClaude AIに添付して使ってください。
以下から目的に合ったプロンプトをコピーして使います。

---

## 基本プロンプト

### シンプルリサーチ
```
「{テーマ}」についてリサーチし、INSTRUCTIONS.mdの形式でJSONを出力してください。
```

### ターゲット指定
```
「{テーマ}」について、{ターゲット}向けの資料用JSONを出力してください。
INSTRUCTIONS.mdの形式・文字数制限に従ってください。
```

## BtoB向けプロンプト

### 提案書
```
「{テーマ}」について、BtoB提案書用の資料JSONを出力してください。
以下の方針で：
- セクション1: 市場課題・背景
- セクション2: ソリューション・アプローチ
- セクション3: 実績・今後の展望
トーンは堅めのビジネス文体。
INSTRUCTIONS.mdの形式・文字数制限に従ってください。
```

### 競合分析
```
「{テーマ}」の競合分析資料用JSONを出力してください。
以下の方針で：
- セクション1: 市場概況
- セクション2: 主要プレイヤー比較（cardsで4社）+ 数値データ（statsで3指標）
- セクション3: 時系列の変遷（timeline）
INSTRUCTIONS.mdの形式・文字数制限に従ってください。
```

### ROI・効果検証
```
「{テーマ}」の導入効果をまとめた資料JSONを出力してください。
以下の方針で：
- セクション1: 導入前の課題
- セクション2: 施策内容（cards）+ 定量効果（stats）
- セクション3: 導入ステップ（timeline）
数値は具体的に。
INSTRUCTIONS.mdの形式・文字数制限に従ってください。
```

## クオリティ向上オプション

以下をプロンプトの末尾に追加すると品質が上がります。
組み合わせ自由。

### データ重視
```
statsの数値は可能な限り実データや信頼できる調査結果に基づいてください。
出典が不明な場合は「推定」と明記してください。
```

### ストーリー性
```
3セクションを「課題提起 → 解決策 → 未来像」のストーリーで構成してください。
closing_quoteは全体を締める印象的な一言にしてください。
```

### 比較・対比
```
cardsは対比構造にしてください（例: 従来手法 vs 新手法、メリット vs デメリット）。
```

### 数字のインパクト
```
statsの数値はインパクトのある見せ方にしてください。
例: 「50%削減」より「1/2に短縮」、「200%増」より「3倍に成長」
```

### 簡潔さ重視
```
各テキストは文字数制限の8割以下に収めてください。
余白がある方がスライドは読みやすくなります。
```

## 応用プロンプト

### テーマ未定（ブレスト）
```
{業界/分野}で今ホットなトピックを3つ提案してください。
それぞれについて、上記JSON形式で資料化した場合のtitle_top/title_mid/title_bottomだけ先に出してください。
選んだらそのテーマで完全なJSONを出力します。
```

### 既存資料のリメイク
```
以下の内容を元に、INSTRUCTIONS.mdの形式でJSONを出力してください。
内容を要約・再構成して3セクションに収めてください。

---
{既存資料のテキストをここに貼る}
---
```

### 英語資料
```
Research "{theme}" and output JSON following INSTRUCTIONS.md format.
All text fields should be in English.
Character limits still apply (count by characters, not words).
```
